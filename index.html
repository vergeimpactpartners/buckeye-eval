Platform Guide
How this platform is structured, what each page is for, and how to use it.
What is EvalCommons?
EvalCommons is the evaluation workspace for the Buckeye EcoVillage Developmental Evaluation, built and managed by Verge Impact Partners. It is a single platform with three distinct pages, each serving a different audience and purpose.
Evaluator
The full workspace for the evaluation team. Manage data collection, run AI sensemaking, track fidelity, and control what residents see. Only accessible to VIP team members.
EHW
A shared page for EVB partners and EHW team members. Shows evaluation aims, learning questions, and sprint progress. Read-only — no editing or publishing from this page.
Village
A resident-facing record of the evaluation in progress. Shows the meeting timeline, BAR/AAR activity records, and emerging themes. Content only appears here when an evaluator explicitly publishes it.
Three URLs, three audiences
Evaluator — buckeye-eval.netlify.app
Full evaluator workspace. All 14 views available. Publish controls visible throughout.
Publish features available:
  • Publish to EHW — pushes learning widgets (title, learning questions, fidelity tracker) to the EHW page
  • Set as default — controls which Village activity opens automatically when residents arrive
  • Publish to Village — makes emerging themes visible on the Village page. Once published, stays published permanently
EHW — buckeye-eval.netlify.app?mode=ehw
Shared view for EVB partners and EHW team. Shows published learning content and sprint progress. Includes a button to switch to the Village page. Read-only — no publish controls.
Village — buckeye-eval.netlify.app?mode=village
Resident-facing page. Shows the meeting timeline, activity records (BAR/AAR), and emerging themes when published. No evaluator controls visible. Always opens with the default activity set by the evaluator.
Set as default:
Each activity has a Set as default button in its header (evaluator view only). Clicking it saves to Drive — when any resident opens the Village link, that activity opens automatically and all others collapse.
Templates
Activity template (Village page)
Every activity on the Village page follows the same four-section structure:
  1. Header bar — activity name, subtitle, Set as default button, collapse toggle
  2. Blue-pale banner — framing question + any context cards (e.g. EVB Way)
  3. White content area — BAR → action bubble → AAR, horizontally scrollable
  4. Magenta themes panel — emerging themes, hidden from residents until published
To add a new activity, say "new BAR/AAR" to Claude — you'll be prompted for the framing question, subcategories, and content for each BAR question before anything is built.
Learning widget template (EHW page)
Learning widgets are the core unit of the evaluator workspace. Each has three sections:
  1. What we are testing — the learning question or hypothesis
  2. How we are testing it — the embedded data collection action
  3. What we are learning — findings as they emerge
Published widgets appear on the EHW page in the same structure. Unpublishing removes them immediately.
Color system
Colors are used consistently across the platform to signal function, not decoration. When adding new elements, use this as a guide:
Blue
Process, structure, navigation, meeting timeline
Magenta
Learning — widgets, questions, findings, emerging themes
Orange
Action — data collection, BAR/AAR activity headers
Midnight
Sensemaking — coding, AI analysis, activity structure
Green
Completion — done meetings, published states
Amber
Up next — upcoming meetings, pending states
Current build status
The platform is live at buckeye-eval.netlify.app. Data syncs to Google Drive via César's account — publish state, fidelity assessments, and stories are shared across all evaluator devices. The voice story widget and AI Sensemaking require HTTPS, which Netlify provides automatically.
● Drive Data syncing to Google Drive — all evaluators see the same state
○ Local Drive temporarily unavailable — data saved locally and will sync when reconnected
How to update the platform
The platform is a single HTML file deployed via GitHub and Netlify. To update:
  1. Make changes with Claude — share the current file and describe what you need
  2. Download the updated file from Claude
  3. Go to github.com → buckeye-evalcommons → index.html
  4. Click the pencil icon → select all (Cmd+A) → paste the new file (Cmd+V)
  5. Scroll down and click Commit changes
  6. Netlify auto-deploys within 60 seconds
Village page continuity. Before uploading any update, confirm that changes to the Village page are intentional. Published content stays published across deploys — the only way to remove it is to explicitly unpublish.
Tip — continuity across sessions
Claude remembers the platform's history, structure, and your preferences across conversations. If starting a new chat, mention this is the EvalCommons platform for Buckeye EcoVillage DE and share the current file — Claude will pick up right where you left off.

Strong connection
Belonging ↔ Governance: Stories about belonging most often arise in governance contexts — specifically in moments of inclusion or exclusion from collective decisions. These themes co-occur in 5 of 6 belonging-coded passages.
Enabling condition
Relational trust → Governance fluency: In all 4 governance-coded passages, relational trust is mentioned as a precondition for effective participation. Trust appears to be upstream of governance readiness.
Tension
Onboarding ↔ Belonging: Onboarding challenges and belonging are appearing in narratives from different cohorts — newer residents name onboarding struggles while longer-term residents describe belonging. This may reflect a developmental arc rather than a fixed split.

Buckeye EcoVillage Developmental Evaluation

          
Welcome to the Project Home

          
This page keeps you connected to the evaluation process — what we're learning, what questions are guiding our inquiry, and how you can share your experience.

        


        
        

          
About This Project

          
The DE aims to track, make sense of, and strengthen how the EcoVillage Buckeye Project operationalizes its theory of change AND demonstrates what the "EHW way", at the center of EVB model looks, feels, and functions like in practice.

        


        
        

          
Evaluation Aims

          

            

              
Aim 1 — Document and Distill the Model

              
Document the key resources, practices, principles, and ways of working that enable or impede the EVB resident-led approach.

            

            

              
Aim 2 — Understand the Difference Made

              
Understand the "difference made" when all partners work in the "EHW way" — with fidelity to resident leadership, village principles, community definitions of resilience and wellbeing, and community-endorsed priorities — and what that difference reveals about the roles, goals, and power of partners in a community-led model.

            

            

              
Aim 3 — Improve the Practice

              
Surface what Phase 1 reveals — for EHW, resident leadership and partners — about the conditions, assumptions, and dynamics expressed in the theory of change, in order to strengthen and adapt resident-led practice that is central to the "EHW way" for the next phase of the EVB work.
